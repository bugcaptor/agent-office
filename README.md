# Agent Office

여러 코딩 에이전트 터미널 세션을 **2D 픽셀 아트 사무실**로 관리하는 데스크톱 앱입니다.

Claude Code 같은 에이전트를 여러 개 돌리면 어느 터미널이 무슨 일을 하고 있었는지 놓치기 쉽습니다. Agent Office는 세션 하나를 사무실에서 일하는 캐릭터로 보여줘, 타이쿤 게임 구경하듯 에이전트들을 한눈에 파악하고 오갈 수 있게 합니다.

## 주요 기능

- **에이전트 1명 = 터미널 세션 1개** — 캐릭터를 클릭하면 터미널이 열립니다. 닫아도 세션은 사라지지 않고 보존됩니다(보이기만 토글).
- **알림** — 에이전트가 응답을 기다리면 캐릭터 머리 위에 느낌표가 뜨고, 화면 오른쪽에 티커 알림이 쌓입니다. 클릭해 터미널을 열면 알림이 지워집니다.
- **머리 위 작업 라벨** — 각 캐릭터 위에 프로젝트·목표·현재 작업을 두 줄로 표시합니다. 로컬 CLI로 요약할 수도 있습니다(옵트인).
- **세션 시간 집계** — 프롬프트 입력부터 응답 완료까지, 세션별 작업/대기 시간을 패널로 보여줍니다(옵트인).
- **캐릭터 일기** — 세션이 끝날 때 각 캐릭터가 자기 성격을 문체 삼아 작업 로그 겸 일기를 씁니다(옵트인). 탭 메뉴 → **일기 보기**로 열람합니다.
- **작업 폴더 탐색** — 탭 메뉴 → **작업 폴더 보기**로 파일 목록·diff·커밋 히스토리·저장소 커밋 로그를 앱 안에서 훑어봅니다.
- **봇 모드 (실험 중)** — 캐릭터를 Gitea 이슈를 스스로 읽고 작업하는 봇으로 전환합니다. 아래 [봇 모드](#봇-모드-실험-중) 참고.
- **픽셀 아트 캐릭터** — 프로필을 만들면 시드 기반 픽셀 캐릭터가 자동 생성됩니다. 인간 외에 엘프·오크·로봇·슬라임 등 아키타입 8종을 지원하며, 직접 그린 스프라이트 업로드(크롭·배경 투명화)나 PixelLab API로 교체할 수 있습니다.
- **사무실 연출** — 세션이 도는 동안 캐릭터는 자리에 앉아 일하고, 한가해지면 탕비실로 가 쉽니다. 밝음·미드나이트·벚꽃 테마 3종을 제공합니다.

## Claude / Codex 연동 (옵트인)

선택적 연동은 **전부 기본 꺼짐**입니다. 첫 실행 동의 화면이나 하단 ⚙ 설정에서 켤 수 있습니다.

- **작업 라벨 요약** — Claude Haiku 또는 Codex `gpt-5.4-mini` 중 하나를 골라, 프롬프트마다 목표/현재를 한 번에 요약합니다. 세션이 실제 작업을 시작하면 목표를 한 번 더 다듬습니다('이슈 40 해결' → 실제 목적으로 구체화).
- **캐릭터 일기** — 요약과 같은 CLI로 세션 종료 시 일기를 자동 생성합니다(수동 '일기 쓰기' 버튼도 그대로).
- **에이전트 관찰 (알림·시간측정)** — 새로 만드는 터미널에 `claude`/`codex`용 훅을 준비해, 알림과 시간측정을 켭니다. `codex resume --last`, 프로필의 시작 명령도 같은 셸 래퍼를 통과합니다.

주의사항:

- 호출은 **선택한 로컬 CLI 계정의 구독 사용량/크레딧을 소비**합니다. 실패해도 다른 provider·모델로 재시도하지 않습니다.
- Agent Office는 **전역 Claude/Codex 설정을 수정하거나 승인을 대신하지 않습니다.** Codex hook은 Codex `/hooks` 화면에서 직접 검토·신뢰해야 합니다.
- 관찰 래퍼는 Windows PowerShell·PowerShell 7·Git Bash·zsh를 지원합니다(WSL 제외). 설정 변경은 **새로 만드는 터미널부터** 적용됩니다.
- 연동이 꺼져 있거나 실패해도 세션 관리·캐릭터·테마 등 나머지 기능은 계속 동작합니다.

## 봇 모드 (실험 중)

> ⚠️ 실험 기능입니다. 동작·범위가 바뀔 수 있습니다.

탭 단위로 캐릭터를 **Gitea 이슈를 스스로 읽고 작업하는 봇**으로 전환합니다. 사람은 봇 모드를 켜고(=승인) 이후엔 이슈 댓글로만 대화합니다. 봇이 PR을 만들면 사람이 머지하고 역할은 끝납니다.

방어선은 두 겹입니다.

1. **로컬 승인** — 반드시 로컬 GUI에서 사람이 탭의 봇 모드를 켜야 발동합니다. 원격에서 켤 방법은 없습니다.
2. **화이트리스트** — 봇이 켜져 있어도 허용된 작성자의 슬래시 명령만 실행합니다(기본값 = `GITEA_TOKEN` 소유 계정). 추가 계정은 GUI에서만 편집합니다.

앱은 **읽기 전용 폴링**만 하고, 접수/진행/완료 댓글과 PR 생성은 에이전트가 직접 합니다. 위협 모델·전체 흐름은 `docs/bot-mode-design.md`를 참고하세요.

## CLI로 조종하기 (옵트인, 2단계 승인)

다른 AI나 스크립트가 실행 중인 Agent Office를 조종할 수 있습니다. 보안 표면이라 **2단계 옵트인**입니다(기본 꺼짐).

1. ⚙ 설정 → **CLI 제어**를 켠다 → 로컬(`127.0.0.1`) 제어 서버가 임의 포트로 뜹니다.
2. **CLI 제어 승인**을 눌러 토큰을 발급한다 → 이때만 토큰이 생기고, 그 전엔 모든 요청이 401입니다(**승인 취소**로 언제든 폐기).

이용하는 쪽은 **주소·포트·토큰을 몰라도 됩니다.** 같은 바이너리의 `ctl` 서브커맨드가 자동으로 찾습니다. `ctl`은 GUI를 띄우지 않고 요청 1건 보내고 종료합니다.

```bash
agent-office ctl status                      # 연결/승인 상태 점검
agent-office ctl list                        # 프로필 + 실행 중 세션
agent-office ctl create reviewer --cwd ~/proj
agent-office ctl send builder "npm test" --enter
agent-office ctl notifications builder --json
agent-office ctl dispose reviewer
agent-office ctl settings set soundEnabled=false
```

설계·위협모델·전체 명령은 `docs/cli-control-design.md`를 참고하세요.

## 사용 팁

### 터미널 탭 우클릭 메뉴

캐릭터를 클릭하면 터미널이 열리고 상단에 세션 탭이 나타납니다. **탭을 우클릭**하면 자주 쓰는 동작이 뜹니다.

- **터미널 재시작** / **이전 세션 이어하기** / **터미널 종료** — 셸·세션을 다시 만들거나 되살리거나 끝냅니다.
- **봇 모드 시작/끄기** — 이 탭을 봇으로 전환합니다([봇 모드](#봇-모드-실험-중) 참고).
- **작업 폴더 보기** — 파일·diff·커밋 로그를 앱 안에서 훑어봅니다.
- **VS Code로 열기** / **OS 터미널로 열기** — 프로필의 시작 폴더(cwd)를 외부 도구로 엽니다(cwd가 있어야 활성화).
- **셸 출력을 에디터로 보기** / **일기 보기** — 세션 출력이나 캐릭터 일기를 봅니다.
- **프로필 편집** — 이름·아키타입·외형·시작 명령어 등을 수정합니다.
- **퇴근** — 세션을 종료하고 캐릭터를 내보냅니다(나중에 다시 소환).
- **캐릭터 삭제** — 세션과 프로필을 완전히 제거합니다.

### 예쁜 캐릭터를 쉽게 만드는 법

시드 자동 생성 스프라이트도 되지만, **프롬프트를 복사해 이미지 생성 AI로 만든 그림을 올리는 것**이 원하는 외형을 얻는 가장 쉬운 방법입니다.

1. 탭 우클릭 → **프로필 편집**.
2. **외모 힌트**·**픽셀아트 의뢰 문구**에 원하는 외형을 적습니다(예: "빨간 망토를 두른 마법사").
3. **픽셀아트 프롬프트 복사**(스프라이트) 또는 **초상 프롬프트 복사**(초상화) 버튼으로 프롬프트를 클립보드에 복사합니다.
4. 이미지 생성 AI에 붙여넣어 그림을 만듭니다.
5. **픽셀아트 업로드** / **이미지 업로드** 버튼으로 올리면 끝(크롭·배경 투명화 지원). `PIXELLAB_API_KEY`가 있으면 **PixelLab로 생성**으로 바로 만들 수도 있습니다.

> 업로드·생성 버튼은 **이미 만든 캐릭터의 프로필을 편집할 때** 나타납니다. 먼저 캐릭터를 만든 뒤 외형을 다듬으세요.

## 실행 방법

### 요구사항

- Node 18+
- Rust 툴체인 ([rustup.rs](https://rustup.rs))
- Tauri v2 기반 — macOS/Windows/Linux 지원

### 개발 서버

```bash
npm install && npm run tauri dev
```

### 릴리즈 빌드

```bash
npm install && npm run tauri build
```

결과물은 `src-tauri/target/release/`에 생성됩니다(macOS `.dmg`, Windows `.msi`/`.exe`, Linux `.deb`/`.AppImage`).

### 테스트

```bash
npx vitest run --dir src                                # 프런트
cargo test --manifest-path src-tauri/Cargo.toml         # 네이티브
npm run typecheck
```

### 선택 환경변수

- `PIXELLAB_API_KEY` — 프로필 편집의 "PixelLab로 생성" 버튼용. 없으면 그 버튼만 비활성화됩니다.

### 폰트 참고

터미널/UI 폰트는 CDN 없이 로컬 번들입니다(Galmuri11, Neo둥근모을 `DungGeunMo`로 대체). 자세한 내용은 `src/renderer/styles/fonts/README.md` 참고.
